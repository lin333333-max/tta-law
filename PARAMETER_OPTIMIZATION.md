# 参数优化完整分析与调整方案

## 📊 当前问题诊断

### 结果分析
```
当前Ma-F1: 35.45%
预期Ma-F1: 45-50%
差距: -10-15%

问题:
1. 低于预期10-15个百分点
2. Mi-F1=57% vs Ma-F1=35%（差距22%）
3. 说明罕见标签仍然预测很差
```

### 根本原因
```
原因1: 学习率2e-5过大
  - 训练数据只有39k（论文163k）
  - 样本少需要更小的学习率
  - 容易过拟合，罕见标签学不好

原因2: Batch size=32偏大
  - 大batch对罕见标签不友好
  - 每个batch很可能不包含罕见标签
  - 梯度更新不够频繁

原因3: Epochs=20不够
  - 罕见标签需要更多训练轮数
  - 降低学习率后需要更多epoch收敛

原因4: 对齐权重base_lambda=1.0偏小
  - 对齐率从92%降到85%
  - 说明对齐约束不够强
```

---

## 🎯 优化方案

### 方案A: 保守优化（推荐）

```python
# config.py 修改

# 核心参数
learning_rate = 1e-5      # 从2e-5降低50%
num_epochs = 30           # 从20增加50%
batch_size = 32           # 保持（已经合理）
warmup_ratio = 0.1        # 保持

# 对齐增强
base_lambda = 2.0         # 从1.0翻倍
```

**理由**:
- 学习率1e-5适合小数据集
- 30 epochs给罕见标签更多学习机会
- batch=32已经是不错的平衡
- 对齐权重加倍提升一致性

**预期效果**: 35.45% → 42-45% (+7-10%)

---

### 方案B: 激进优化（效果更好）

```python
# config.py 修改

# 核心参数
learning_rate = 5e-6      # 降低75%（更小更稳）
num_epochs = 40           # 增加到40
batch_size = 16           # 减半（对罕见标签更友好）
warmup_ratio = 0.15       # 增加warmup

# 对齐增强
base_lambda = 3.0         # 增加3倍
```

**理由**:
- 5e-6超小学习率，避免震荡
- 40 epochs充分训练
- batch=16更频繁梯度更新，罕见标签每个batch出现概率更高
- 更长warmup保证稳定
- base_lambda=3.0强化对齐

**预期效果**: 35.45% → 48-52% (+13-17%)

**代价**: 训练时间翻倍（3天 vs 1.5天）

## 📋 我为你选择的方案

### **混合方案（方案A+部分方案B）**

```python
learning_rate = 5e-6      # 方案B的学习率（关键！）
num_epochs = 40           # 方案B的轮数
batch_size = 16           # 方案B的batch size
warmup_ratio = 0.15       # 方案B的warmup
base_lambda = 3.0         # 方案B的对齐权重
```

**为什么这样选择？**

1. **learning_rate = 5e-6**（最关键）
   ```
   理论计算:
   LR ∝ √(DataSize / DataSize_论文)
   LR ≈ 2e-5 × √(39k/163k) ≈ 2e-5 × 0.49 ≈ 1e-5
   
   考虑罕见标签 → 再降低50% → 5e-6 ✓
   ```

2. **batch_size = 16**（重要）
   ```
   小batch优势:
   - 更频繁的梯度更新（每epoch从1227步→2454步）
   - 罕见标签在每个batch出现概率更高
   - 梯度噪声帮助泛化
   
   代价: 训练时间翻倍（可接受）
   ```

3. **num_epochs = 40**
   ```
   学习率降低4倍(2e-5→5e-6) → 需要更多epoch收敛
   20 × 2 = 40 ✓
   ```

4. **base_lambda = 3.0**
   ```
   对齐率: 92% → 85% (下降7%)
   说明对齐约束太弱
   增大到3.0，预期提升到90%+
   ```
